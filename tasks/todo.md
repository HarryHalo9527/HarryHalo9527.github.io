# 代码审查与修复计划

## 目标
- 对项目源码执行全量代码审查，覆盖根目录主站、`个人主页v2.0/` 镜像、Cloudflare Worker 与关键配置文件。
- 修复审查中发现的安全与稳定性问题，并针对手机端聊天卡住、报告生成较慢做体验优化。

## 检查项
- [x] 盘点需要纳入审查的源码目录，排除依赖与二进制资源
- [x] 审查前端页面结构、资源引用与可运行性风险
- [x] 审查前端脚本中的逻辑缺陷、异常处理与同步一致性问题
- [x] 审查样式文件中的兼容性、冗余与潜在维护风险
- [x] 审查 Cloudflare Worker 与部署配置中的安全性和稳定性问题
- [x] 汇总问题，按严重程度输出结论
- [x] 用环境变量替换 Worker 中硬编码密钥
- [x] 将 Worker 的内存限流改为持久化限流
- [x] 修复前端基于 `innerHTML` 的注入风险
- [x] 修复 TURN 测评重复点击导致的重复计分
- [x] 优化移动端聊天请求超时与失败兜底
- [x] 优化评估报告生成时长与超时回退体验
- [x] 校验根目录与 `个人主页v2.0/` 的同步一致性

## 验证方式
- 通过静态阅读源码、全文检索与必要的本地命令检查交叉验证。
- 对重复目录仅在差异存在时单独指出，避免重复结论。

## 结果复盘
- 已确认根目录源码与 `个人主页v2.0/` 对应文件当前无差异，审查结论可视为同时适用于两份代码。
- 发现 4 类高价值问题：密钥硬编码、Worker 限流设计失效、LLM 输出未做转义导致 XSS 风险、TURN 问卷存在重复计分窗口。
- 依赖目录 `cloudflare-worker/node_modules/` 未纳入审查范围；未执行浏览器端自动化测试，仅完成静态检查与语法级验证。
- 手机端截图显示聊天消息停留在“思考中”，初步判断是请求无超时控制或代理响应不稳定，需增加超时和更快的失败回退。
- 报告生成慢与移动网络环境相关，需同时收敛 token、降低温度/长度并加上明确的本地回退路径。
- 已完成前端稳定性修复：聊天/报告请求加入超时与重试，失败时会快速回退，不再长时间悬挂。
- 已完成双目录同步校验：根目录 `scripts/` 与 `个人主页v2.0/scripts/` 当前无差异。
- Worker 代码已改为读取 `GEMINI_API_KEY` secret，并支持基于 `RATE_LIMIT_KV` 的持久化日限流；若部署环境未绑定 KV，将以降级模式运行。
- 已新增可分享的短链接入口：`/v1/`（传统静态版）与 `/v2/`（AI 动态版）。
- 已在 v2.0 页面加入“版本入口”卡片，并在 v1.0 页面补充“切换到 v2.0 AI 版”入口，便于访客双向切换。
